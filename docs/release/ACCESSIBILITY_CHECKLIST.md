# Accessibility checklist

- [x] Reduced-motion CSS support exists.
- [x] Boot loading state has a status label.
- [ ] Keyboard-only audit for all desktop screens.
- [ ] Visible focus audit.
- [ ] VoiceOver and TalkBack real-device audit.
- [ ] Dynamic Type / 200% zoom audit.
- [ ] Automated and manual contrast audit.
- [ ] 44×44 pt touch-target audit.
- [ ] Form error association and live-region audit.
- [ ] PDF annotation non-visual alternative.
- [ ] No hover-only actions across all breakpoints.

Accessibility is not release-verified until unchecked items have evidence.
