# Release evidence

This directory separates automated evidence, simulator evidence, manual test plans, and credential-dependent blockers. A document saying “pass” must name the command or environment used. “Blocked” and “not tested” are not failures hidden as success.

Start with `FINAL_PREFLIGHT_SUMMARY.md`, `PRODUCTION_READINESS.md`, `KNOWN_LIMITATIONS.md`, and `RELEASE_CHECKLIST.md`.
