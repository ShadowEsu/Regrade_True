# Regrade documentation

Only public-repository-safe material belongs here. Never add credentials, real
student work, access tokens, private school contracts, or workstation paths.

- `release/` — setup, QA, store, and release gates
- `planning/` — product, architecture, design, and historical plans
- `training/` — AI evaluation and supported-content guidance
- `integrations/` — connector implementation and authorization
- `legal/` — draft policies requiring professional review
- `security/` — threat model, rules, and privacy controls
- `internal/` — public-safe templates only; use ignored `*.local.md` for private notes

Current release truth is in `release/PRODUCTION_READINESS.md`. Older planning
documents are historical and must not be treated as proof that a feature works.
